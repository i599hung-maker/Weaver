# 占驗派紫微斗數排盤網站 — 設計規格

日期：2026-07-08　狀態：已與命主確認設計，實作中

## 目標

依文墨天機安星碼 **S5VoG**（占驗派，天乙上人一系）的排盤規則做一個排盤網站：

- 輸入出生資料 → 排出本命盤（12 宮格）
- 一排按鈕切換**十年大限**、一排按鈕切換**流年**（操作同文墨天機）
- 排盤結果與文墨天機（S5VoG 設定）**逐宮一致**才算完成
- 分析功能為第二階段（見文末），排盤定盤通過後才開工

UI 參考 tianfu-ai.com 的深色盤面：宮內主星彩色直列、宮干支右上、大限歲數左下、宮名右下、疊宮小徽章（大命／年財…）；四化用中文單字徽章「祿權科忌」，不用英文字母。

## 架構

純前端靜態網站：Vite + React + TypeScript。星曜計算用 iztro 2.5.8，外加兩層：

```
BirthInput → solarTime(真太陽時) → 晚子時換日 → iztro bySolar(fixLeap=true)
          → patch層(天馬/截空/旬空) → ZhanyanChart → UI
```

- `src/engine/zhanyanConfig.ts` — `astro.config()`：十天干四化表全數明寫鎖死
- `src/engine/solarTime.ts` — 真太陽時（經度差＋均時差），可開關，預設開
- `src/engine/cast.ts` — 入口：時辰換算、晚子時視為次日（≥23:00 → 日期+1、子時）、呼叫 iztro、套 patch、算子斗
- `src/engine/patch.ts` — 占驗派安星修正
- `src/components/` — PalaceGrid / Palace / CenterInfo / HoroscopeBar / BirthForm

## 定盤反推出的占驗派規則（三命例交叉驗證）

三張文墨天機定盤：
1. 命例一 男 1996-05-12 23:40（晚子時）→ 丙子年三月廿六子時、水二局、命宮壬辰
2. 命例二 女 1994-12-02 02:43 丑時 → 甲戌年十月三十丑時、火六局、命宮甲戌
3. 命例三 女 1969-03-11 11:50 午時 → 己酉年正月廿三午時、金四局、命宮壬申

與 iztro 預設輸出比對，**僅兩處不同**，需 patch：

### 天馬依月支（驛馬三合）
寅午戌月→申、申子辰月→寅、巳酉丑月→亥、亥卯未月→巳。
月支取安命宮所用之農曆月（fixLeap 調整後）：`月宮 = 命宮 + 時辰index`（寅起正月）。
驗證：辰月→寅✓、亥月→巳✓（依年支會錯放申）、寅月→申✓（依年支會錯放亥）。

### 截空旬空占驗派排法（單星制）
- **截空**：截路空亡兩宮（甲己年申酉、乙庚午未、丙辛辰巳、丁壬寅卯、戊癸子丑）中，取**支陰陽與年干陰陽相同**之宮，安截空單星。驗證：丙→辰✓、甲→申✓、己→酉✓。
- **旬空**：生年干支所在旬的空亡兩支中，取**支陰陽與年干陰陽相異**之宮，安旬空單星。驗證：丙子年→酉✓、甲戌年→酉✓、己酉年→寅✓。
- 實作：移除 iztro 的截路、空亡、旬空三星，按上述規則補回截空、旬空各一。

### 子斗（斗君）
文墨中宮「子斗」＝**子年斗君**：由子宮逆數（生月−1）、再順數生時。
`子斗支 = (0 − (月−1) + 時辰index) mod 12`。驗證：戌✓辰✓午✓。
流年斗君 = 子斗 + 流年支偏移。

### 其餘全部與 iztro 預設一致（已驗證）
魁鉞、祿存羊陀、火鈴（年支+時支）、天空、地空地劫、天使天傷、命主身主、
五行局、大限起訖與順逆（陽男陰女順）、小限起宮與男順女逆、農曆換算、四化。

### 真太陽時
文墨顯示並採用真太陽時定時辰。實作：均時差近似式＋經度修正（預設經度 121.0，可輸入），
可開關（預設開）。三命例均未跨時辰界，待遇到邊界案例再校精度。

## 第二階段：分析功能（2026-07-08 實作完成）

架構決策（與命主確認）：**規則引擎＋AI 混合**。斷應期完全由程式計算（確定性、可測試）；
解讀文字由 Claude 生成。本地使用**不需 API key**：dev 伺服器（vite middleware
`server/analyzePlugin.ts`）以子行程呼叫本機已登入的 `claude -p --model sonnet`（headless，
移除 CLAUDECODE 環境變數），若設 `ANTHROPIC_API_KEY` 則改走 API（部署用）。

### 規則引擎（`src/analysis/`）

1. **主題**：本命=命宮、財運=財帛、事業=官祿、愛情=夫妻；各取**三方四正**
   （本宮＋對宮＋三合兩宮）主星輔星、亮度、生年四化 → `facts.ts`。
2. **大限疊宮**：大X宮支 = 大限命宮支 −宮序偏移（命0 兄1 妻2 子3 財4 疾5 遷6 友7 官8 田9 福10 父11）。
3. **斷應期**（`trigger.ts`，逐大限×流年）：
   - **流命引動**：流年命宮支 == 主題大限宮支（事業→大官、財→大財、愛情→大妻、本命→大命）；
     **災宮引動**：== 大疾支（課程定義確認：災宮=大疾）。
   - **疊星引動**（需流年層參與；生年×大限記大限備註）：
     **同星相疊**＝生年/大限/流年四化有兩層以上化同一顆星；
     **四化交會**＝兩層以上四化落同一宮。權重：忌疊忌3、含忌2、其餘1。
   - 疊星命中歸屬主題：命中宮位落在主題三方四正 → 該主題，否則「整體」。
   - 每筆命中皆產生**原因鏈**文字。
4. 測試：`__tests__/trigger.test.ts` 定盤一手算案例（甲午限大官戌、2018 戊戌事業引動、
   2021 辛丑災宮、2026 丙戌忌疊忌等）。

### AI 解讀

- `buildPrompt()`：命主資訊＋主題三方四正＋斷應期重點命中（權重≥2 或流命/災宮、
  大限≤82歲起）＋解讀要求（只准用給定事實、先本質再大限再重點年、每年原因鏈、
  800~1200字繁中 markdown）。
- 效能實測：sonnet headless 約 3~5 分鐘/主題，伺服器逾時 600 秒；結果以
  localStorage 快取（key=命例+主題），避免重複生成。
- UI（`AnalysisPanel.tsx`）：四主題頁籤；斷應期時間軸（每大限一列，年份卡標示
  流命引動/災宮/同星相疊/四化交會，預設只看重點、可切全部；點年份跳轉盤面該流年）；
  AI 解讀按鈕＋文字區。規則表不依賴 AI，離線可用。

### 課程尚未提供規則、待補

三代論、太歲入卦、小限秘術、命相同參（斷應期補充技法）；災宮是否另含其他宮位。

## 競品學習：八字人格天賦報告 App（2026-07-09，截圖在 /LifePath/A參考）

該 App 的內容配方（值得借用）：

1. **天賦類型命名**：給命主一個人格原型稱號（例：「獨立思考的真理探索家」）＋格局標籤（印重局）＋角色插圖。一句話身分認同，記憶點極強。
2. **解讀關鍵詞雲**：優點／缺點兩欄關鍵詞，字級大小＝權重（例：智慧、評估力、學習力 vs 控制慾、固執、猶豫不決）。一眼掃完整個人。
3. **最強天賦**：多段「你是⋯」第二人稱身分句（你是理想主義的組織者／分析型的問題解決專家），肯定語氣。
4. **閃光點（1-2-3）→ 弱點（1-2-3）→ 潛力提升建議**：每個弱點都配對一條可執行的成長建議（關注情緒、尋找平衡、給予別人質疑的權利）——正是「知命改命」的結構。
5. **全程「認識自己」框架**：叫人格報告不叫算命，無吉凶字眼，正向心理學語氣；缺點寫得誠實直接（傲慢、工作狂）反而增加可信度。
6. 商業面：1對1諮詢入口＋邀請朋友測（病毒循環）。

可落地到占驗紫微：星曜→關鍵詞對照表（規則引擎可產關鍵詞雲）；命宮主星＋格局→天賦類型命名表；
報告增加「你是⋯」天賦段與「弱點→提升建議」配對段。

注意：該 App 晚子時用當日（陰曆三月廿五、日柱己酉），與占驗派「視為次日」不同，屬流派差異，勿抄。
